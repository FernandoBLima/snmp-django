1 - O que é SNMP?

O SNMP é um conjunto de protocolos onde máquinas na rede local podem ser designadas como gerentes ou agentes. A máquina gerente pode transmitir mensagens SNMP via TCP ou UDP para os agentes e processar sua resposta. A máquina gerente pode, ainda, receber mensagens espontâneas dos agentes, as quais são denominadas Traps. Desta forma, a máquina gerente é capaz de facilmente monitorar o funcionamento de diversos agentes na rede local, tanto de forma ativa como de forma passiva. O SNMP costuma estar encapsulado em pacotes de software voltados à gerência de redes, mas não é incomum que o gerente necessite escrever um script para auxiliar na construção de uma solução para seu problema específico. Para identificar um objeto da rede, o SNMP faz uso de da MIB (Management Information Base), a qual consiste em uma árvore hierárquica que separa os tipos de informação a serem buscados, e o OID (Object Identifier), que identifica um tipo determinado de objeto em um agente alvo. Com objeto, queremos dizer um componente como a placa de rede, a memória, o cooler, etc. A MIB e os OIDs são conhecidos a priori e tabelados, de maneira que o usuário do SNMP apenas necessita configurar os agentes e consultar as tabelas para montar a solução para seu problema de gerência. Isto torna o SNMP especialmente útil no caso de uma rede com muitos agentes, como, por exemplo, um datacenter. Em um datacenter real, é muito custoso monitorar manualmente cada componente de cada máquina da rede, então podemos configurar Traps SNMP que enviam alertas para o gerente no caso de falha em determinado componente, evitando assim riscos e cortando gastos com manutenção. Uma das maiores vantagens do SNMP reside no fato de que o processamento e armazenamento de dados são delegados para a máquina gerente, o que faz com que se economize esforço de processamento.

4 - Nossa solução

A partir do script base em Python que envia uma mensagem SNMP para um agente, desenvolvemos uma interface gráfica utilizando o framework django. Ela recebe as opções do usuário em um formulário e as envia como entrada para o script no back-end, recebendo o retorno do agente e escrevendo-o no front-end para o usuário.
